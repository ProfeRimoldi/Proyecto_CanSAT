# Documentation – Proyecto CanSat

Esta carpeta contiene la documentación técnica principal del proyecto, incluyendo arquitectura general del sistema, criterios de diseño, requerimientos, validaciones y evolución del desarrollo.

Aquí se concentra la trazabilidad técnica global entre hardware, firmware y validaciones experimentales.

---

## Contenido esperado

* arquitectura general del sistema
* requerimientos funcionales
* máquina de estados de misión
* plan de validación y ensayos
* decisiones de diseño
* criterios de seguridad
* cronograma de desarrollo
* informes técnicos
* documentación de revisiones

---

## Objetivo

Mantener una metodología de desarrollo profesional orientada a proyectos electrónicos aeroespaciales, permitiendo reconstruir decisiones técnicas, justificar revisiones y facilitar futuras iteraciones del sistema.

La documentación debe acompañar al hardware y firmware, no aparecer únicamente al final del proyecto.

---

## Relación con otras carpetas

* `hardware/` → diseño fuente y fabricación de PCB
* `firmware/` → desarrollo de software embbido
* `docs/` → documentación técnica global del sistema

---

## Nota importante

Cada cambio importante de hardware o firmware deberás tener su respaldo técnico correspondiente en esta carpeta.
